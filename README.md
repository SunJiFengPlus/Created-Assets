#### 用户上传数据较大, 每次部署时都要上传好久, 根据用户上传数据的特点决定分工程部署.
#### 这样每次更新迭代时代码时只需要把核心工程打 war 包后上传到服务器即可.
---
### 技术点
* AOP 参数验证
* MyBatis 自定义的通用枚举类型转换器
* REST API
* 定时任务清理临时文件
* 全局异常处理封装返回相应的错误码与简短错误信息
* Java Json Web Token 鉴权
* 外部 API: 短信、人机验证
* XSS 拦截 TODO 
* 规范的测试类
